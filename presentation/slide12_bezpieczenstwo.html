Zastosowane zabezpieczenia

Dostęp

  • Entra ID — logowanie i role (pracownik / przełożony / dyrektor)
  • JWT w API Management — każde wywołanie API z tożsamością

Sekrety

  • Key Vault — klucze i parametry poza kodem
  • Managed Identity — usługi Azure łączą się bez haseł w repozytorium

Dane

  • Szyfrowanie SQL i Blob (w spoczynku i w tranzycie)
  • WORM — niezmienny audyt decyzji w archiwum

Nadzór

  • App Insights — logi i śledzenie żądań API
  • Reguła progu w backendzie i bazie — niezależnie od UI
